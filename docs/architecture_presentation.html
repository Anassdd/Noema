Noema · architecture

One question, two memories, every claim cited

The expert's knowledge lives in two stores built from the same documents: a contextual RAG base for precise passages, and a knowledge graph for entities and how they relate. Every question searches both.

The answer path
💬

Question

The user asks. A quick routing check decides if the knowledge base is needed at all — small talk is answered directly.

route · gpt-5.4-mini

Contextual RAG base

Document chunks, each enriched with its context, embedded as vectors — finds the exact passages that talk about the question.

Graph base

Entities and facts extracted from the same documents — finds knowledge connected across documents, not just similar wording.

⚖

Fuse & check

The two result lists are merged into one ranking, and the evidence is graded: is this enough to answer? If not, the search widens.

grading · gpt-5.4-mini
✒

Grounded answer

The answer is written ONLY from the retrieved evidence, and cites its sources — document and passage.

answer · gpt-5.4-mini
✔

Self-check

A final faithfulness check: does every claim trace back to the sources? If not, the answer is retried (up to twice) before being shown.

verify · gpt-5.4-mini
How the two memories are built (ingestion)
📄

Documents

PDFs and papers uploaded on the ingestion page — the single source both memories are built from.

🔍

Parse

Each page is read by a strong vision model — text, tables and formulas come out clean, with page-level provenance kept.

parsing · gpt-5.4

→ Contextual RAG base

Chunks get a short context blurb, then are embedded (text-embedding-3-large) into the vector store.

→ Graph base

Entities and relationships are extracted (gpt-5.4) page by page into the temporal knowledge graph — the one ✦ Dream maintains.

gpt-5.4 — the strong model: PDF parsing & graph extraction (memory quality)
gpt-5.4-mini — routing, grading, answering, self-checks
text-embedding-3-large — multilingual embeddings (French corpus ready)
