⚗ Bench

Testing the memories on real, human-written questions

A one-press benchmark page: pick a public dataset, build the memories once, ask the same questions through four configurations, and get a finished, comparable report. No LLM-generated gold — the questions and answers were written by human researchers (QASPER, NAACL 2021).

The pipeline
1

Dataset

Public research-paper corpus with questions written and answered by human annotators, with the evidence marked.

QASPER — 1,585 NLP papers, human Q&A
2

Prepare

Whole papers are selected up to a token cap; only the human questions whose papers made the cut are kept as gold.

Pilot: 16 papers · ~100k tokens · 46 questions
3

Build once

The corpus is ingested into BOTH memories exactly once — fingerprinted, so re-running never rebuilds or re-pays. Pause/resume safe.

The graph lands in a save — explorable in 3D after
4

Ask ×4

Every question is asked through the four configurations below — same generator model, same prompts, only the memory changes.

46 questions × 4 configs = 184 answers
5

Judge

An independent judge from a different model family compares each answer to the human gold. Plus word-overlap scores and evidence recall.

Judge = Gemini — can't favor its own answers
6

Report

A finished, versioned deliverable: verdict, per-question-type slices, worst-failure gallery, costs — comparable run to run.

Downloadable markdown, kept per run
The four configurations — and the pilot result

Closed book

No memory at all — measures what the model already knows. This is the honest floor every method must beat.

28%floor

Contextual RAG

Vector base only — precise passages.

65%+37 over floor

Graph

Knowledge graph only — connected facts.

33%+5 over floor

Hybrid

Both, fused — the app's default mode.

59%+31 over floor
Each memory wins somewhere: RAG is strongest on factual questions (69% vs hybrid's 56%), but on synthesis questions — “explain how / why” — hybrid overtakes it (50% vs 37.5%). The graph adds understanding, the vectors add precision.
