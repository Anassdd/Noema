✦ Dream

The memory doesn't just grow — it maintains itself

One press runs a self-maintenance cycle over the knowledge graph — like the brain consolidating memories during sleep. It keeps the memory bounded and high-quality as documents keep arriving, instead of letting it explode.

The cycle
1

Analyze

Snapshot the graph and sanity-check the read. If the graph can't be read consistently, Dream refuses to touch it.

Never self-maintains on a bad read
2

Plan

Decide which passes are actually worth running, and say why. Nothing to improve → it stops right there.

“3 likely duplicates · 5 stale facts”
3

Dedupe

Merge entities that split over time. Fuzzy matches are confirmed by an LLM before any merge; all facts and provenance are repointed onto the survivor.

“BNPP” + “BNP Paribas” → one entity
4

Forget

Facts superseded by newer information are archived out of active retrieval after a grace period; disconnected leftover entities are pruned.

Archived, never deleted — history stays visible
5

Consolidate · off for now

Refresh community summaries — big-picture groupings over the graph. Disabled pending a graph-library fix; dedupe + forget carry the load.

Re-enabled after the library upgrade
Every pass runs inside a safety net
⛨

Checkpoint

Before anything changes, the whole graph is copied. The original is always one step away.

▶

Run the pass

One attempt, no retry loops — a maintenance step must never be able to spiral.

✔

Verify

Source documents intact · no current fact lost · no unexplained node removals · retrieval probes still answer.
